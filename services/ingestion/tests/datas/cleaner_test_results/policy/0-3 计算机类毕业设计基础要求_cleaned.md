# **计算机类毕业设计基础要求**

本要求为计算机类专业毕业设计的统一基准规范，旨在保障课题可行性与成果质量，所有选题及执行过程需严格遵循。

## **一、成果形式要求**

毕业设计需提交“技术成果+文档成果”两类核心材料，具体规范如下：1. **技术成果** ：需提交可独立运行的完整技术成果，拒绝“仅理论设计无实际实现”。根据选题类型区分要求：开发类选题（系统/工具/平台）：提交可编译、可运行的源代码（含详细注释，注释率不低于 30%）、编译运行说明文档、核心功能演示视频（时长 3-5 分钟）；

2. 算法研究类选题：提交可复现的实验代码、数据集（脱敏处理）、实验日志及结果可视化文件；
3. 硬件结合类选题：除上述软件材料外，需提交硬件实物（或仿真模型）及硬件连接、调试说明。
4. **文档成果** ：提交符合学术规范的毕业设计论文，字数≥5000 字（不含代码、图表附录），需包含“摘要、研究背景、技术方案、实现过程、测试结果、总结展望”六大核心章节，引用文献≥10 篇（其中近 3 年中英文文献占比不低于 50%）。
## **二、时间约束要求**

毕业设计整体周期为 4-6 个月，选题需满足“阶段可控、周期适配”原则，具体要求：1. 选题需拆解为“需求分析（1-2 周）、方案设计（2-3 周）、开发/研究（8-10 周）、测试优化（2-3 周）、论文撰写（3-4 周）”五个阶段，各阶段任务需在周期内可完成；

2. 禁止选择“超范围宏大课题”，如“通用人工智能对话系统开发”“跨行业大数据平台构建”等需长期资源投入的课题，优先选择“聚焦单一场景、目标明确”的细分课题；
3. 中期检查（第 2-3 个月）需完成核心功能开发或研究框架搭建，未达要求者需调整选题。
## **三、可行性核心要求**

选题需充分考量本科生资源可及性与能力适配性，以下情形视为“不可行选题”，不予通过：- 资源不可获取：需专属昂贵硬件（单套成本≥5000 元）、非公开涉密数据或行业专属权限，且无法通过学校实验室、校企合作渠道解决；

- 技术门槛过高：依赖本科生未接触的深度专业技术（如大规模集成电路设计、底层操作系统开发），且无导师全程专项指导；

- 成果不可验证：无法通过实验、测试或场景演示验证技术有效性（如无明确评价指标的“智能优化算法研究”）。
